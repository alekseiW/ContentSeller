# Функциональная спецификация платформы GuideHub

## 1. Авторизация и регистрация

### Страница регистрации
- Email, пароль, подтверждение пароля
- Или регистрация через Google / Telegram OAuth
- Поле "Имя автора" (отображается на публичных страницах)
- Поле "Никнейм" — формирует URL: `platform.ru/u/nickname`
- Toggle: "Я самозанятый" (влияет на дальнейший онбординг платежей)

### Страница входа
- Email + пароль
- "Забыли пароль?" → восстановление по email
- Войти через Google / Telegram

### Онбординг после регистрации (3 шага, опционально)
1. "Что вы будете продавать?" — выбор из 4 категорий: Гайды, Мини-курсы, Шаблоны, Другое
2. "Где ваша аудитория?" — выбор: Telegram, Instagram, YouTube, VK
3. "Подключить оплату сейчас?" —_skip или подключить ЮKassa/Prodamus_

## 2. Дашборд (Личный кабинет)

### Структура
**Header:** Логотип | Мои продукты | Аналитика | Настройки | Аватар автора (dropdown)

**Верхние виджеты (3 карточки):**
- Выручка за 30 дней (₽) с % изменения к прошлому периоду
- Количество продаж за 30 дней
- Конверсия (% покупок от просмотров)
- Клик по карточке → переход в детальную аналитику

**Секция "Мои продукты":**
- Кнопка "+ Новый продукт" (primary)
- Вид карточек: обложка + название + статус (активен/черновик) + мини-статистика (просмотры, продажи, выручка)
- Сортировка: по дате, по выручке, по просмотрам
- Фильтр: Все / Активные / Черновики
- Hover на карточку: иконки "Редактировать", "Открыть страницу", "Дублировать", "Удалить"

**Пустое состояние:**
- Иллюстрация + текст "Создайте первый продукт" + кнопка

**Правая колонка (сайдбар или секция):**
- "Быстрые действия": Создать гайд, Создать мини-курс, Загрузить файл
- "Последние продажи": 3-5 последних транзакций
- Туториал: "Как начать" — 3-шаговый чек-лист

## 3. Редактор контента

Это ядро продукта. Свободный WYSIWYG-редактор в стиле Obsidian/Notion, но без жёсткой блочной метафоры. Ощущение — как пишешь документ, а не собираешь блоки.

### Общий Layout
- **Верхняя панель:**
  - Кнопка "← Назад" (возврат на дашборд)
  - Название документа (кликовый, можно переименовать inline)
  - Статус автосохранения ("Сохранено")
  - Кнопки: "Предпросмотр", "⚙️ Настройки продукта", "Опубликовать"
  - Аватары collaborators (на будущее)

- **Левая боковая панель (сворачиваемая):**
  - Оглавление — список заголовков/секций документа
  - При навигации — скролл к нужной части
  - Drag & drop заголовков = реорганизация разделов

- **Центральная область — редактор:**
  - Бесконечное полотно с прокруткой
  - Ощущение пустого листа, как в Google Docs
  - Текст пишется сразу, без "создания блоков"
  - Ширина контента ~720px, выравнивание по центру

- **Правая панель (появляется при работе с медиа/модулями):**
  - Контекстные настройки выбранного элемента
  - Для изображения: размер, выравнивание, подпись, скругление углов
  - Для видео: превью, autoplay, зацикливание
  - Для модуля: параметры конкретного модуля

### Ввод и форматирование текста
- Свободный ввод текста, как в любом текстовом редакторе
- Выделение текста → всплывающий тулбар форматирования:
  - **Bold**, *Italic*, ~~Зачёркнутый~~, `Код`, Ссылка
  - Заголовок H1/H2/H3
  - Цитата
  - Маркированный / нумерованный список
  - Цвет текста (акцентный)
- Markdown shortcuts: `# Заголовок`, `**bold**`, `> цитата`, `- список`, `---` разделитель
- Enter создаёт новую строку/абзац естественно
- Tab = отступ, Shift+Tab = убрать отступ

### Slash-команды и вставка модулей
- Ввод `/` в любом месте текста → всплывающее меню с командной палитрой
- Меню фильтруется по мере ввода
- Доступные модули/команды:

**Контент:**
- `/heading` — заголовок (H1/H2/H3)
- `/quote` — блок цитаты с вертикальной полосой
- `/divider` — горизонтальный разделитель
- `/callout` — выделенный блок с иконкой (info, warning, tip, idea)
- `/code` — блок кода с подсветкой синтаксиса
- `/toggle` — сворачиваемый блок (аккордеон)
- `/columns` — двухколоночная раскладка
- `/button` — кликабельная кнопка внутри контента

**Медиа:**
- `/image` — вставка изображения (загрузка с устройства / URL / AI-генерация)
- `/gallery` — сетка/карусель изображений
- `/video` — встраивание видео (YouTube, Vimeo, прямая ссылка, загрузка)
- `/embed` — эмбед стороннего контента (Figma, Google Maps, Twitter, Telegram-пост)
- `/audio` — аудиофайл / подкаст
- `/file` — вложение (PDF, документ для скачивания)

**Структурные:**
- `/checklist` — чек-лист с интерактивными чекбоксами
- `/table` — таблица
- `/card` — карточка с заголовком, текстом и иконкой
- `/rating` — блок с звёздами/рейтингом
- `/pricing` — блок с ценой (для сравнения тарифов внутри гайда)
- `/cta` — призыв к действию

**AI-команды:**
- `/ai-write` — написать текст с помощью ИИ (тема, тон, длина)
- `/ai-improve` — улучшить выделенный текст
- `/ai-shorten` — сократить выделенный текст
- `/ai-expand` — развернуть мысль
- `/ai-translate` — перевести выделенный текст
- `/ai-cover` — сгенерировать обложку раздела

### Drag & Drop и перемещение модулей
- При наведении на любой модуль появляются:
  - Иконка "⠿" (6 точек) слева — для перетаскивания
  - Меню "⋯" справа — для действий с модулем
- Drag & drop: перетаскивание модуля в другое место документа
- Меню модуля: Дублировать, Удалить, Превратить в..., Настройки, Скрыть до оплаты
- Визуальный drag-индикатор — линия-подсветка между секциями

### Inline-редактирование медиа
- **Изображения:**
  - Клик по изображению → появляется рамка с ресайз-хендлами
  - Ресайз: маленький / средний / большой / полная ширина
  - Выравнивание: лево / центр / право
  - Скругление углов: без / лёгкое / сильное
  - Подпись под изображением (inline)
  - Hover-эффект: зум при наведении (опционально)

- **Видео:**
  - Автопревью с YouTube/Vimeo
  - Загрузка собственного видео (до 500 МБ)
  - Настройки: autoplay, loops, controls, poster image
  - Placeholder перед загрузкой

- **Embeds:**
  - Автоматический превью сторонних ссылок
  - Поддерживаются: YouTube, Vimeo, Google Maps, Figma, CodePen, Twitter, Telegram-посты
  - Embed рендерится прямо в потоке текста

- **Файлы:**
  - Отображаются как компактная карточка с иконкой типа файла
  - Название, размер
  - Кликабельны для скачивания
  - Можно пометить как "доступно только после оплаты"

### AI-инструменты (inline)
- Иконка ✨ появляется при:
  - Выделении текста → "Улучшить", "Сократить", "Развернуть", "Перефразировать"
  - Пустой строке → "Продолжить" (AI дописывает)
- AI-панель (выдвижная справа): "Спросить ИИ" — чат-интерфейс
  - "Напиши введение к гайду про..."
  - "Добавь больше деталей в этот раздел"
  - "Создай чек-лист на основе текста выше"
- AI-генерация обложки: описание → 4 варианта → выбор

### Автосохранение
- Сохранение каждые 2–3 секунды при изменениях
- Визуальный индикатор: "Сохранено" / "Сохранение..." / "Не сохранено"
- История версий (в Pro): доступ к предыдущим версиям документа

## 4. Мастер создания продукта (Wizard)

Опциональный упрощённый flow для тех, кто не хочет работать с полноценным редактором сразу.

### Шаг 1: Источник контента
- "Вставьте текст" — большое поле ввода
- "Загрузить файл" — .txt, .docx, .pdf, .md
- "Создать с нуля в редакторе"

При вставке текста или загрузке файла:
- AI-обработка: разбиение на секции, выделение заголовков
- Предпросмотр структуры (сворачиваемая панель)
- Кнопка: "Перейти в редактор для доработки" или "Продолжить настройку"

При AI-генерации:
- Поле "О чём ваш гайд?"
- Генерация структуры → показ → одобрение/редактирование → переход в редактор

### Шаг 2: Оформление
- Выбор шаблона (6-8 вариантов): Минимал, Журнал, Тревел, Обучение, Чек-лист, Портфолио, Блог, Техно
- Акцентный цвет: палитра 8 цветов
- AI-генерация обложки: описание → 4 варианта
- Live-preview справа: как будет выглядеть публичная страница

### Шаг 3: Настройка продукта
- Название (если не указано ранее)
- AI-описание (editable)
- Цена (₽) с подсказкой AI
- Toggle: "Бесплатное превью" — ON/OFF
  - При ON: настройка, сколько контента видно (1 секция, 2 секции, %)
- Toggle: "Мини-курс" — разбить на уроки
  - Если ON: каждая секция H1 = отдельный урок
  - Настройка порядка уроков
  - Drip-контент: выпуск по расписанию (Pro)
- Toggle: "Скачиваемый файл" — прикрепить PDF/документ после оплаты

### Шаг 4: Публикация
- URL: `platform.ru/u/nickname/product-name`
- Редактируемый slug
- Кнопка "Опубликовать"
- После публикации:
  - Ссылка с кнопкой "Скопировать"
  - Кнопки "Поделиться в Telegram", "Поделиться в VK"
  - QR-код
  - Кнопки: "Открыть страницу" | "Назад в дашборд" | "Редактировать"

## 5. Публичная страница продукта

Страница, которую видит потенциальный покупатель. Оптимизирована для конверсии.

### Структура

**Header:**
- Логотип платформы (маленький, кликабельный)
- Кнопка "Создать свой гайд" (правый верхний угол)
- Без навигации — чистая фокус-страница

**Hero-секция:**
- Полноширинная обложка (фото или градиент с генерацией паттерна)
- Поверх или после: название гайда (H1)
- Подзаголовок (описание)
- Строка автора: аватар + имя
- Мета: "× min чтения" (автоматически по объёму)

**Что внутри (секция):**
- 4-6 буллитов с иконками-чек-марками
- AI-генерированные на основе контента
- Автор может редактировать

**Превью контента:**
- 2-3 секции из начала гайда (если автор включил превью)
- Красиво отрендеренный контент: текст, заголовки, изображения, списки
- В конце — визуальный "занавес": градиентное размытие в белый
- Текст на занавесе: "Получите полный доступ, чтобы прочитать остальное"

**Блок покупки (sticky-бар на мобильных):**
- Цена: `490 ₽` (крупно)
- Кнопка "Купить гайд" (primary, large)
- Под ней: иконки платёжных систем (СБП, Мир, Visa, MC)
- "Мгновенный доступ после оплаты"
- Гарантия: "30 дней возврат" (опционально)

**Отзывы (если есть):**
- 2-3 карточки отзывов
- Аватар, имя, текст, дата, рейтинг (звёзды)

**Секция "Бонусы" (если есть):**
- Дополнительные материалы, доступные вместе с покупкой
- Список бонусов с иконками

**Секция "От автора":**
- Аватар + имя автора
- Био/описание
- Ссылки на соцсети
- Другие продукты автора

**Footer:**
- "Powered by GuideHub"
- Ссылка "Создайте свой гайд"

### После оплаты (страница трансформируется):
- Кнопка "Купить" → "Читать гайд"
- Занавес с размытием снимается
- Весь контент доступен для чтения
- Появляется кнопка "Скачать PDF" (если включено)
- Внизу: "Создать свой продукт" — баннер для посетителей платформы

## 6. Страница оплаты (Checkout)

Отдельная страница или модальное окно при клике "Купить".

### Структура
- Логотип платформы (верхний левый угол)
- Левая колонка:
  - Превью продукта (обложка маленькая)
  - Название
  - Цена
  - "От автора: [имя]"
- Правая колонка:
  - Email покупателя
  - Способ оплаты:
    - Банковская карта (форма: номер, срок, CVV)
    - СБП (QR-код для сканирования)
    - ЮMoney
    - СБПэй
  - Кнопка "Оплатить [490 ₽]"
  - Чекбокс: "Согласен с условиями оферты"
  - "Безопасная оплата через [партнёр]"
- Мобильная версия:
  - Вертикальный layout
  - Sticky кнопка оплаты внизу
  - Свайп между способами оплаты

### После успешной оплаты:
- Экран "Оплачено ✓"
- Кнопка: "Перейти к гайду"
- Email с ссылкой на гайд отправлен
- Инструкция: если закрыл страницу — найди ссылку в почте

### После неуспешной оплаты:
- "Оплата не прошла"
- Кнопка "Попробовать снова"
- Кнопка "Другой способ оплаты"

## 7. Страница чтения гайда (после покупки)

Полноценная страница с полным доступом к контенту.

### Layout
- Без header платформы — только контент
- Верхняя полоска (тонкая):
  - Название гайда
  - Кнопка "← Назад к продукту"
  - Кнопка "Скачать PDF"
- Контент: полный гайд, красиво отрендеренный
- Оглавление слева (sticky, скроллирующееся)
- Широкие изображения на всю ширину контейнера
- Плавный скролл, плавное раскрытие аккордеонов
- Прогресс-бар чтения сверху (тонкая полоска)

### Функционал
- Возможность вернуться к превью-версии
- Кнопка "Поделиться" (копировать ссылку, реферальная ссылка)
- Кнопка "Оставить отзыв"
- Если мини-курс: переключение между уроками (prev/next)
- Отметка уроков как "прочитано"

## 8. Аналитика

### Верхние метрики (4 карточки):
- Выручка за период (₽) + %
- Количество продаж + %
- Просмотры страницы + %
- Конверсия (%) + %

### Переключатель периода:
- 7 дней / 30 дней / 90 дней / Всё время / Произвольный

### Графики:
1. **"Выручка по дням"** — линейный график
2. **"Просмотры и покупки по дням"** — двойной линейный график
3. **"Конверсия"** — числовое значение с трендом
4. **"Воронка продаж"** — горизонтальная воронка:
   - Просмотры страницы → Просмотр превью → Начало оплаты → Успешная оплата
   - С % конверсии на каждом этапе

### Таблица "Последние продажи":
- Дата и время
- Продукт
- Сумма
- Статус (успешно/возврат)
- Email покупателя
- Пагинация

### Топ продуктов:
- Рейтинг продуктов по выручке
- Мини-бар: относительная высота баре

### География (Pro):
- Карта с регионами покупателей
- Разбивка по городам

### Источники трафика (Pro):
- Telegram
- Прямой
- VK
- Другие
- С % и количеством

## 9. Настройки

### Профиль
- Аватар (загрузка, кроп)
- Имя автора (публичное)
- Био (описание, отображается на публичных страницах)

- Ссылки на соцсети: Telegram, Instagram, YouTube, VK
- Превью публичного профиля (справа)

### Платежи
- Статус подключения: "✅ Подключено" или "Не подключено"
- Кнопка: "Подключить ЮKassa" или "Подключить Prodamus"
- Статус самозанятости: проверено / ожидает
- История выплат:
  - Дата, сумма, статус
  - Кнопка "Вывести средства"
- Настройка минимальной суммы вывода
- Налоговый статус: самозанятый / ИП / не указан

### Уведомления
- Toggle: "Email при покупке"
- Toggle: "Новый отзыв/комментарий"
- Toggle: "Еженедельный отчёт на почту"
- Toggle: "Push-уведомления" (если есть PWA)
- Toggle: "СМС при крупной покупке" (Pro)

### Тарифный план
- Текущий план: Free / Pro
- Кнопка: "Обновить до Pro — 790 ₽/мес"
- Сравнение тарифов в таблице
- История оплат
- Кнопка: "Отменить подписку"

### Аккаунт
- Сменить email
- Сменить пароль
- Экспорт данных (все продукты, статистика)
- Удалить аккаунт (с подтверждением)
- Кнопка выхода

## 12. Системные страницы

### Страница-заглушка (404)
- "Страница не найдена"
- Кнопка "На главную" и "Создать свой гайд"

- "Этот продукт больше недоступен"
- Ссылка на профиль автора

## 13. Не-функциональные требования

### Производительность
- Страница гайда загружается < 2 сек
- Редактор: debounce автосохранения 2 сек
- Lazy loading изображений и медиа
- CDN для статики

### Безопасность
- HTTPS обязательно
- Защита от копирования (опционально для авторов Pro): отключение ПКМ, водяные знаки с email покупателя, ограничение на печать
- Rate limiting на checkout
- Защита от XSS в контенте гайдов
- JWT аутентификация

### Мобильная адаптация
- Mobile-first приоритет для публичных страниц
- > 70% трафика из Telegram — мобильные
- Sticky-бар покупки на мобильных
- Редактор — адаптирован для планшетов (не обязательна полная мобильная версия для создания, но чтение — обязательно)

### SEO
- Публичные страницы — индексируемые
- Open Graph мета-теги для шеринга в Telegram/VK
- Sitemap для продуктов
- ЧПУ URL

### Масштабируемость
- Мульти-арендность (multi-tenant)
- Возможность масштабирования на 10k+ авторов
- Поддержка нескольких платёжных шлюзов
